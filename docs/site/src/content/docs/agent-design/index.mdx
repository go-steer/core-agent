your agent actually behave the way you want. Prescriptive content
rather than reference — what patterns work, what failure modes to
watch for, how to make the model use subagents and `agentic_*`
wrappers efficiently.

## Reading order

If you're new to `core-agent`, walk these in order — each builds on
the prior. Operators who've used `core-agent` for a while can pick
whichever covers the specific friction they're hitting; the pages
cross-link generously.
