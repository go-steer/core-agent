**Use when:** you want the model to think through the change *first*,
commit that thinking to an artifact a human can review, and only then
start editing. Common in code-review flows, migrations,
change-management environments, and unattended runs where a bad plan
is cheaper to catch than a bad diff.

## How it works

`core-agent`'s permission gate has a **plan-first** posture: the model
must call the `record_plan` tool with a written plan before any
mutating tool (`write_file`, `edit_file`, `delete_file`, `bash` with
non-read-only commands) is allowed. The gate refuses those tool calls
until the plan is on file.

<Aside type="tip">
Turn it on with `permissions.RequirePlanArtifact: true` in the library
API, or `--require-plan` on the CLI.
</Aside>
